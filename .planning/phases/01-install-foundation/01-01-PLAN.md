Create three files:

**scripts/setup.sh** — Idempotent install/upgrade script (per D-01, D-02, D-03, D-05, D-06, D-07):
```bash
#!/usr/bin/env bash
set -euo pipefail

INSTALL_DIR="$(cd "$(dirname "$0")/.." && pwd)"
VENV="$INSTALL_DIR/.venv"
SERVICE_NAME="animaya"
SERVICE_DIR="$HOME/.config/systemd/user"
SERVICE_FILE="$SERVICE_DIR/$SERVICE_NAME.service"
DATA_DIR="$HOME/hub/knowledge/animaya"
VERSION_FILE="$HOME/.animaya-version"
CURRENT_VERSION="0.1.0"
```

Sections in order:

1. **.env handling (D-01):**
   - If `$INSTALL_DIR/.env` exists: source it, check `TELEGRAM_BOT_TOKEN` and `CLAUDE_CODE_OAUTH_TOKEN` are non-empty, exit 1 with error message if either missing.
   - If `.env` missing: `read -rp "Telegram bot token: " TELEGRAM_BOT_TOKEN`, then `read -rs -p "Claude OAuth token: " CLAUDE_CODE_OAUTH_TOKEN` (silent for token), echo newline, write .env with those two vars plus `DATA_PATH=$DATA_DIR`. Immediately `chmod 600 "$INSTALL_DIR/.env"`.

2. **Node.js check (D-03):**
   - `if ! command -v node &>/dev/null; then echo "WARNING: Node.js not found. Claude Code SDK requires Node.js. Install it before running Animaya."; fi`
   - Do NOT auto-install Node.js (could break Claude Code's own Node). Just warn.

3. **Python venv (D-02):**
   - `[[ -d "$VENV" ]] || python3 -m venv "$VENV"`
   - `"$VENV/bin/pip" install -q -e "$INSTALL_DIR"`
   - Always runs pip install (handles upgrades on re-run per D-07).

4. **Hub knowledge dir (D-06):**
   - `mkdir -p "$DATA_DIR"`

5. **Migration (D-08 discretion):**
   - Read `$VERSION_FILE` if exists. If version differs from `$CURRENT_VERSION`, log "Migrating from $old to $CURRENT_VERSION". For 0.1.0, no migration steps needed — just update the version file.
   - `echo "$CURRENT_VERSION" > "$VERSION_FILE"`

6. **Systemd service (INST-02, INST-03):**
   - `mkdir -p "$SERVICE_DIR"`
   - `cp "$INSTALL_DIR/systemd/animaya.service" "$SERVICE_FILE"`
   - Test systemd user mode: `if ! systemctl --user status &>/dev/null 2>&1; then echo "WARNING: systemd user mode not available. You may need to install the service as a system service."; exit 0; fi`
   - `loginctl enable-linger "$USER" 2>/dev/null || true`
   - `systemctl --user daemon-reload`
   - `systemctl --user enable "$SERVICE_NAME"`
   - `systemctl --user restart "$SERVICE_NAME"`
   - Final message: `echo "Animaya installed and running. Check status: systemctl --user status animaya"`

Make executable: the file must have `#!/usr/bin/env bash` shebang.

**run.sh** — Systemd ExecStart wrapper (D-04, INST-04):
```bash
#!/usr/bin/env bash
set -euo pipefail

INSTALL_DIR="$(cd "$(dirname "$0")" && pwd)"

# INST-04: Sanitize Claude Code env vars to prevent SDK subprocess hangs
unset CLAUDECODE
unset CLAUDECODE_EXECUTION_ID

# Load environment
set -a
source "$INSTALL_DIR/.env"
set +a

# Activate venv and run
exec "$INSTALL_DIR/.venv/bin/python" -m bot
```

Note: use explicit venv python path instead of `source activate` for reliability.

**systemd/animaya.service** — User service unit (INST-03):
```ini
[Unit]
Description=Animaya AI Assistant
After=network.target

[Service]
Type=simple
WorkingDirectory=%h/animaya
ExecStart=%h/animaya/run.sh
Restart=on-failure
RestartSec=5
StandardOutput=journal
StandardError=journal
SyslogIdentifier=animaya

[Install]
WantedBy=default.target
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input -> .env file | Interactive token prompts written to disk |
| .env file -> run.sh | Secrets sourced into process environment |
| systemd -> run.sh | Service manager executes wrapper script |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | .env file | mitigate | chmod 600 .env immediately after creation in setup.sh |
| T-01-02 | Information Disclosure | Token prompt | mitigate | Use `read -rs` (silent) for CLAUDE_CODE_OAUTH_TOKEN input |
| T-01-03 | Elevation of Privilege | ExecStart path | mitigate | Use %h expansion in service file, no hardcoded paths |
| T-01-04 | Tampering | CLAUDECODE env injection | mitigate | unset CLAUDECODE + CLAUDECODE_EXECUTION_ID in run.sh before sourcing .env |
</threat_model>

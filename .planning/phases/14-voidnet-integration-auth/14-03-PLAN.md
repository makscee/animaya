## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| external proxy (voidnet-api) → Edge middleware | Every proxied request carries attacker-controllable headers; middleware must re-verify per request |
| Edge middleware → downstream route handlers | Once voidnet verified, session cookie is signed by AUTH_SECRET; downstream treats it as it would any Telegram session |
| error response → client | Fixed-shape JSON; no diagnostic detail leakage |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-01 | Tampering (replay) | middleware voidnet branch | mitigate | Verifier runs per request (no cache); ±60s window enforced inside verifier |
| T-14-02 | Spoofing (forged sig) | middleware verify + JWT mint | mitigate | Secret never leaves server; HMAC verified via crypto.subtle before JWT mint; failure short-circuits to 401 before encode() runs |
| T-14-03 | Elevation of Privilege (owner bypass) | middleware owner enforcement | mitigate | 403 returned by verifier before JWT mint; no silent fallback to Telegram path on voidnet failure (CONTEXT contract) |
| T-14-04 | Info Disclosure (error leak) | middleware error branch | mitigate | Only `{error: result.error, code: result.code}` forwarded; verifier emits only fixed strings; no stack traces via `NextResponse.json` |
| T-14-05 | Spoofing (JWT salt mismatch → downstream session null) | encode() call | mitigate | salt value derived from same `isProd` branch as getToken cookieName at middleware.ts:132-134; Plan 01 roundtrip smoke test locks interop; Pitfall 1 covered |

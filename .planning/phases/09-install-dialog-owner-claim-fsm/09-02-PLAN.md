**Part A: Add pairing code functions to telegram_bridge_state.py (D-9.6, D-9.8)**

Add these functions to `bot/modules/telegram_bridge_state.py`:

1. `def generate_pairing_code(module_dir: Path) -> tuple[int, dict]` -- Per D-9.6:
   ```python
   import hmac, hashlib, secrets, os
   from datetime import datetime, timezone, timedelta

   code = secrets.SystemRandom().randint(100000, 999999)
   salt = secrets.token_hex(16)
   key = os.environ.get("SESSION_SECRET", "").encode()
   digest = hmac.new(key, (salt + str(code)).encode(), hashlib.sha256).hexdigest()
   expires = (datetime.now(timezone.utc) + timedelta(minutes=10)).isoformat()
   state = {
       "claim_status": "pending",
       "owner_id": None,
       "pairing_code_hash": digest,
       "pairing_code_salt": salt,
       "pairing_code_expires": expires,
       "pairing_attempts": 0,
   }
   write_state(module_dir, state)
   return code, state
   ```
   Returns the plaintext code (for one-time display) and the written state dict. The plaintext code is NEVER written to disk.

2. `def verify_pairing_code(candidate: str, state: dict) -> bool` -- Per D-9.8/D-9.9:
   - Check expiry: parse `state["pairing_code_expires"]` as ISO datetime, compare to `datetime.now(timezone.utc)`. If expired, return `False`.
   - Check attempt cap: if `state.get("pairing_attempts", 0) >= 5`, return `False`.
   - Compute HMAC: `key = os.environ.get("SESSION_SECRET", "").encode()`, `salt = state.get("pairing_code_salt", "")`, `expected = hmac.new(key, (salt + candidate).encode(), hashlib.sha256).hexdigest()`.
   - Return `hmac.compare_digest(expected, state.get("pairing_code_hash", ""))`.

3. `def check_expiry(state: dict) -> dict` -- Utility to auto-transition expired pending codes:
   - If `state.get("claim_status") == "pending"` and code is expired (TTL check), set `claim_status = "unclaimed"`, clear pairing fields (`pairing_code_hash`, `pairing_code_salt`, `pairing_code_expires` to `None`, `pairing_attempts` to `0`).
   - Return the (possibly modified) state dict. Caller must write it if changed.

Add imports at module top: `import hmac, hashlib, secrets` and `from datetime import datetime, timezone, timedelta`.

**Part B: Bridge claim handler in telegram.py (D-9.9)**

In `bot/bridge/telegram.py`, add a new handler function and register it:

1. Add function `async def _claim_handler(update: Update, context: ContextTypes.DEFAULT_TYPE) -> None`:
   - Get message text: `text = (update.message.text or "").strip()` if `update.message` else return.
   - If `not text.isdigit() or len(text) != 6`: return (not a pairing code, fall through).
   - Get module_dir from `context.bot_data.get("module_dir")`. If None, return.
   - Import and call: `from bot.modules.telegram_bridge_state import read_state, write_state, verify_pairing_code`
   - Read state: `state = read_state(module_dir)`
   - If `state.get("claim_status") != "pending"`: return (not in pairing mode).
   - Increment attempts: `state["pairing_attempts"] = state.get("pairing_attempts", 0) + 1`, `write_state(module_dir, state)`.
   - If `not verify_pairing_code(text, state)`:
     - If `state["pairing_attempts"] >= 5`: set `state["claim_status"] = "unclaimed"`, clear pairing fields, `write_state(module_dir, state)`, reply `"Too many incorrect attempts. Click Regenerate to get a new code."`.
     - Else: reply `f"Incorrect code. {5 - state['pairing_attempts']} attempt(s) remaining."`.
     - Return.
   - On success: `state.update({"claim_status": "claimed", "owner_id": update.effective_user.id, "pairing_code_hash": None, "pairing_code_salt": None, "pairing_code_expires": None, "pairing_attempts": 0})`, `write_state(module_dir, state)`.
   - Reply: `"Ownership claimed. You are the owner of this bot."`.
   - Raise `ApplicationHandlerStop` to prevent further handler processing.

2. Register handler in `build_app()`: add `app.add_handler(TypeHandler(Update, _claim_handler), group=-2)` BEFORE the existing `_owner_gate` handler at group=-1. Import `ApplicationHandlerStop` from `telegram.ext`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Telegram user -> Bridge handler | 6-digit code from untrusted sender |
| Dashboard poll -> claim-status API | Plaintext code must not leak in poll responses |
| Dashboard generate -> browser | One-time code display (acceptable, ephemeral) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-06 | Elevation of Privilege | Bridge claim handler | mitigate | 5-attempt cap per code window; `hmac.compare_digest` timing-safe comparison (D-9.8, D-9.9) |
| T-09-07 | Elevation of Privilege | Expired code reuse | mitigate | TTL check in `verify_pairing_code` + auto-transition in `check_expiry` on every poll (D-9.8) |
| T-09-08 | Information Disclosure | claim-status poll | mitigate | Poll endpoint returns dashes "------" not the actual code; plaintext only shown once at generation time |
| T-09-09 | Spoofing | Pairing code enumeration | mitigate | 6 digits = 900,000 codes; 5 attempts per window = negligible brute-force probability; code expires in 10 min |
| T-09-10 | Tampering | state.json on disk | accept | Single-user LXC system; filesystem access = full control anyway |
</threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Dashboard user -> API | Untrusted token input crosses from browser to backend |
| Backend -> Telegram API | Outbound request with user-supplied token |
| API response -> Browser | Token must never appear in response body |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Information Disclosure | GET /api/modules | mitigate | redact_bridge_config strips token, returns has_token: bool only (D-9.15) |
| T-09-02 | Information Disclosure | Log output | mitigate | Never log token value; log only username from getMe response and token[:4] pattern if debug needed (D-9.16) |
| T-09-03 | Spoofing | POST /install | mitigate | Validate token via Telegram getMe before persisting; reject invalid tokens with no state written (D-9.2) |
| T-09-04 | Denial of Service | POST /install | accept | Single user system; no rate limiting needed on install endpoint |
| T-09-05 | Information Disclosure | Input field | mitigate | Token input uses type="password" to prevent shoulder surfing (UI-SPEC Component 1) |
</threat_model>

    Create `bot/modules/supervisor.py` per D-8.1/D-8.2 + RESEARCH.md Architecture
    Pattern 2. Full contract:

    - `class Supervisor` with private `_handles: dict[str, object]` and
      `_runtime_entries: dict[str, str]` (name → dotted path, so stop_all can re-import).
    - `async def start_all(self, ctx: AppContext) -> None`:
      - Calls `read_registry(ctx.data_path)`.
      - For each entry: skip if `runtime_entry` is None or absent.
      - Emit `module.starting` via `ctx.event_bus("info", "supervisor", f"starting {name}")`.
      - `mod = importlib.import_module(entry["runtime_entry"])`.
      - `handle = await mod.on_start(ctx, entry.get("config", {}) or {})`.
      - Store `self._handles[name] = handle` and `self._runtime_entries[name] = entry["runtime_entry"]`.
      - Emit `module.started`.
      - On exception: `logger.exception(...)`, emit `module.errored`, mark registry entry `state="errored"` via a helper that reads/writes registry atomically, and `continue` (D-8.2 exception policy).
    - `async def stop_all(self) -> None`:
      - Iterate `reversed(list(self._handles.items()))`.
      - Emit `module.stopping`.
      - `mod = importlib.import_module(self._runtime_entries[name])`; `await mod.on_stop(handle)`.
      - Emit `module.stopped`.
      - On exception: `logger.exception(...)`, continue.
      - After loop: clear both dicts.
    - `def get_handle(self, name: str) -> object | None`: returns `self._handles.get(name)`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| manifest.json → Python import | `runtime_entry` string controls which module is imported; platform-authored but still needs namespace validation to prevent arbitrary module execution if manifest is ever user-supplied. |
| registry.json (filesystem) → supervisor | Supervisor reads runtime_entry from registry entries. If registry is tampered, supervisor would import attacker-chosen module. |
| AppContext passed to modules | Modules receive data_path, stop_event, event_bus. Frozen dataclass prevents mutation, but modules can still misuse data_path. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-02 | T (Tampering) / E (Elevation) | `ModuleManifest.runtime_entry` → `importlib.import_module` | mitigate | Task 1 adds pydantic `field_validator` rejecting paths not matching `^bot\.[a-z][a-z0-9_]*(\.[a-z][a-z0-9_]*)+$`. Only `bot.*` namespace allowed. Blocks `os.system`, `subprocess`, etc. |
| T-08-03 | D (DoS) / availability | Supervisor start_all with one bad module | mitigate | Task 2 wraps each module's on_start in try/except; failure logged + event emitted + state marked errored in registry; boot continues. Other modules unaffected. |
| T-08-06 | I (Info Disclosure) | AppContext.event_bus emits to log | accept | event_bus takes (level, source, message) — no token/secret path by design. Module authors must not pass config values. Low risk (internal API). |
| T-08-07 | T (Tampering) | registry.json read by supervisor | accept | Registry is under `~/hub/knowledge/animaya/` — same trust boundary as the bot process. Phase 8 does not introduce new attack surface; v1.0 already treats filesystem as trusted. |
</threat_model>

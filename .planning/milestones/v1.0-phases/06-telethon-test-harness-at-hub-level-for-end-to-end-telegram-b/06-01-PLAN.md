The Animaya bot replies with this lifecycle (from `bot/bridge/telegram.py`):
1. `_send_status()` sends an initial placeholder message (often "тАж") тАФ THIS CAN
   ARRIVE WITHIN MILLISECONDS of the user's send. Event handlers MUST be
   registered BEFORE the send to avoid missing it.
2. `_stream_text()` edits that same message as Claude tokens arrive, throttled
   by `_STREAM_MIN_INTERVAL = 0.5s` and `_STREAM_MIN_CHARS = 30`.
3. `_on_tool_use()` may replace the status message with a new `_send_status()`
   when a tool runs after text was already streamed тАФ so more than one message
   can appear for a single logical reply.
4. `_finalize_stream()` does a final edit with `md_to_html(reply)`; if the text
   exceeds `TG_MAX_LEN = 4096`, it deletes the status and sends sequential
   `reply_text` chunks instead.

Implications for driver lifecycle:
- Listener registration MUST happen BEFORE `send_to_bot` returns тАФ otherwise
  the early `_send_status` placeholder can slip through unhandled. This is why
  the driver splits listener start from reply wait.
- Must subscribe to BOTH `events.NewMessage(from_users=bot_id)` and
  `events.MessageEdited(from_users=bot_id)`.
- Must track every bot message id seen during the window and return the
  concatenation of their final texts (handles the tool-use / long-reply
  multi-message case).
- Must reset a settle timer on every new-or-edited event; when the timer
  expires (default 2.0s, > `_STREAM_MIN_INTERVAL`), treat the reply as final.
- Animaya sends with HTML parse_mode (`<b>`, `<i>`, `<code>`, `<pre>`, `<a href>`).
  Telethon's `message.raw_text` strips formatting entities and is the
  preferred source; as a belt-and-suspenders measure, strip any residual
  `<tag>` sequences with a final `re.sub(r"<[^>]+>", "", text)` pass so
  substring assertions like `"pong"` match regardless of wrapping tags.

Telethon API contracts the driver depends on (resolve current syntax via
context7 `telethon` if anything below drifts):
- `from telethon import TelegramClient, events`
- `client = TelegramClient(session_path, api_id, api_hash)`
- `await client.start(phone=...)` тАФ interactive if no session
- `entity = await client.get_entity(username)` тАФ username without `@` works
- `await client.send_message(entity, text)` тАФ returns Message
- `client.add_event_handler(handler, events.NewMessage(from_users=entity))` and
  `client.remove_event_handler(handler, event_type)` тАФ used so the driver
  controls registration/teardown explicitly (decorator form does not expose a
  clean removal handle).
- Events expose `event.message.id`, `event.message.text`, `event.message.raw_text`

```python
@dataclass
class Listener:
    """Handle returned by start_listening. Owns registered handlers + state."""
    client: "TelegramClient"
    bot_entity: object
    bot_username: str
    # internal state:
    messages: "OrderedDict[int, str]"     # msg_id -> latest text (entity-stripped)
    activity: asyncio.Event                # set on each new/edited bot event
    event_count: int                       # total events observed
    last_event_at: float | None            # loop.time() of last event, or None
    _handlers: list                        # (callback, event_type) pairs for teardown

    async def __aenter__(self) -> "Listener": ...
    async def __aexit__(self, *exc) -> None: ...   # calls self.close()
    async def close(self) -> None: ...             # remove_event_handler for each pair


async def start_listening(bundle) -> Listener:
    """
    Register NewMessage + MessageEdited handlers on bundle.client filtered to
    bundle.bot_entity. MUST be called BEFORE send_to_bot so the bot's initial
    _send_status placeholder cannot race past an unregistered handler.
    Returns a Listener usable as `async with` (auto-closes handlers on exit).
    """


async def send_to_bot(bundle, text: str) -> "telethon.tl.custom.Message":
    """Send `text` to the configured bot. Returns the sent Message."""


async def wait_for_reply(
    listener: Listener,
    timeout: float = 60.0,
    settle: float = 2.0,
) -> str:
    """
    Consume events from an already-started Listener. Returns the final
    consolidated reply text once `settle` seconds have elapsed with no new
    or edited message from the bot. Raises asyncio.TimeoutError with a
    context-aware message on timeout.
    """


def assert_contains(text: str, needle: str) -> None:
    """Raise AssertionError if `needle` not in `text`."""


async def resolve_bot_entity(client, username: str):
    """Thin convenience wrapper around client.get_entity for external callers."""
```

Implementation rules:

1. **Listener state** is an `OrderedDict` keyed by `event.message.id` so the
   final return preserves insertion order across multi-message replies.

2. **`start_listening`** builds two callbacks (one for `NewMessage`, one for
   `MessageEdited`), each of which:
   - Extracts text preferring `event.message.raw_text`, falling back to
     `event.message.text or ""` (raw_text is entity-stripped by Telethon).
   - Applies `re.sub(r"<[^>]+>", "", text)` as a belt-and-suspenders pass so
     any residual HTML from Animaya's `md_to_html` output is stripped before
     storage. Assertions thus see the rendered text the user sees.
   - Writes into `listener.messages[event.message.id]`.
   - Increments `listener.event_count`, sets `listener.activity`, updates
     `listener.last_event_at = asyncio.get_event_loop().time()`.
   Registers both via `bundle.client.add_event_handler(cb, event_type)` and
   records each `(cb, event_type)` in `listener._handlers` for teardown.

3. **`Listener.close()`** iterates `_handlers` and calls
   `self.client.remove_event_handler(cb, event_type)` for each, inside a
   `try/except` that logs but never raises (teardown must be idempotent).
   `__aexit__` calls `close()`.

4. **`wait_for_reply`** algorithm (no handler registration happens here тАФ
   handlers are already live from `start_listening`):
   ```
   start = loop.time(); deadline = start + timeout
   while True:
       remaining = deadline - loop.time()
       if remaining <= 0:
           if listener.event_count == 0:
               raise asyncio.TimeoutError(
                   f"Timeout after {timeout}s with no events from "
                   f"@{listener.bot_username} тАФ bot may be offline, "
                   f"blocked, or the username is wrong."
               )
           raise asyncio.TimeoutError(
               f"Timeout after {timeout}s тАФ {listener.event_count} events "
               f"arrived but reply never settled for {settle}s."
           )
       try:
           await asyncio.wait_for(listener.activity.wait(), timeout=remaining)
           listener.activity.clear()
       except asyncio.TimeoutError:
           continue  # outer deadline check handles the real timeout
       # At least one event has fired. Wait for settle window.
       while True:
           idle = loop.time() - listener.last_event_at
           if idle >= settle:
               break
           # Race: either settle elapses or another event arrives.
           try:
               await asyncio.wait_for(listener.activity.wait(),
                                       timeout=settle - idle)
               listener.activity.clear()
           except asyncio.TimeoutError:
               break
           if loop.time() >= deadline:
               break
       # Settled.
       return "\n\n".join(listener.messages.values()).rstrip()
   ```

5. **Do NOT** perform any teardown inside `wait_for_reply` тАФ the Listener
   context manager owns handler removal.

# Plan: revert continue_conversation gate + reset-dev wipes ~/.claude/projects

## Goal

Undo the `continue_conversation=not in_bootstrap_mode` gate introduced by
260416-l1z. It broke onboarding mid-flow because EVERY query inside
BOOTSTRAP.md mode ran fresh — bot forgot its own prior turns. Replace it
with a cleaner approach: `continue_conversation=True` always; reset-dev.sh
wipes `~/.claude/projects/` so a fresh operator reset also means a fresh
Claude Code SDK session transcript. Prior conversation memory only leaks
across resets if a user doesn't run the reset script — which is never
during fresh-install testing.

## must_haves

- `bot/claude_query.py::build_options` always passes `continue_conversation=True`.
- No branch on `_read_bootstrap()` presence for the continue flag. Other behaviors
  of `_read_bootstrap()` (BOOTSTRAP injection, locale substitution) UNCHANGED.
- The INFO log added by l1z ("continue_conversation disabled") is removed.
- `scripts/reset-dev.sh` wipes `~/.claude/projects/` after stopping the service
  and before starting it.
- `tests/test_claude_query_bootstrap.py` — 3 tests added by l1z that asserted
  `continue_conversation=False` when BOOTSTRAP.md exists are DELETED or rewritten
  to assert `True` unconditionally. Keep remaining bootstrap injection + locale
  tests green.
- ruff clean, pytest green.
